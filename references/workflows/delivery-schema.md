# Standard PPT Delivery Schema

这是 PPT Director 与编程 Agent 之间的核心接口。进入代码生成前，必须把内容整理成此格式。

## Schema

```text
【PPT信息】
- 标题：
- 副标题：
- 受众：
- 目的：
- 时长：
- PPT类型：业务PPT / 酷炫视觉PPT
- 风格：
- 评审专家：
- 核心信息：
- 输出方式：PPTX / Marp / python-pptx代码 / 手动排版材料包

【素材清单】
- 文档：
- 数据：
- 图片：
- Logo/品牌资产：
- 需要AI生成的图片：

【逐页内容】
第1页 | 类型：封面
- 标题：
- 副标题：
- 要点：
- 数据/图表需求：
- 视觉需求：
- 推荐页型：
- 演讲备注：

第2页 | 类型：目录 / 内容页 / 数据页 / 架构图 / 关系图 / 结束页
- 标题：
- 要点：
- 数据/图表需求：
- 视觉需求：
- 推荐页型：
- 演讲备注：

【附录/备选页】
- 可能被追问的问题：
- 可加入的支撑材料：
```

## Rules

- 每页标题必须是观点句，不是主题词。
- 每页要点默认 3-5 条，每条尽量不超过 20 个汉字。
- 每页只服务一个核心信息。
- 数据页标题必须说明“数据在说什么”。
- 视觉需求要明确到图表、示意图、图标、图片、二维码或纯文字。
- 如需代码生成，必须给出图片路径或占位符路径。
